# Workflow Rules

## 开始前

- 先用 `rg` / `rg --files` 查找相关代码、测试、配置和现有模式。
- 先阅读 `AGENTS.md`，再按任务类型阅读本目录对应规则。
- 不确定业务含义时，先查 contract、测试、数据库 schema、现有调用，再提问。
- 不要重写、删除或回滚用户已有改动，除非用户明确要求。

## 实现中

- 遵循现有目录结构和命名风格。
- 新增依赖前先确认现有工具链不能解决问题。
- 修改共享 contract 时，必须同步检查前端调用、后端 handler、测试和文档。
- 不允许在前端临时补一个类型来绕过 contract。
- 不允许在后端返回 contract 未定义的字段。

## 完成前

- 优先运行 `pnpm verify`。
- 如果改动范围较小，可以运行更聚焦的 `pnpm typecheck`、`pnpm lint`、`pnpm test` 或相关 package 脚本。
- 无法运行检查时，必须说明原因、影响范围和剩余风险。
